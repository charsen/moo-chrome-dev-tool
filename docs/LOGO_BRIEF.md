# moo-chrome-dev-tool · Logo 设计 Brief

> 这份 brief 用于把 logo 设计需求交付给设计师。前置已经走完方向取舍（鹰图腾保留、实心剪影、3/4 侧、深圆底 badge、金色 reticle 眼睛），设计师只需在此基础上把鹰的解剖学画准、把视觉做精。

---

## 1. 背景（必读，影响所有视觉决策）

- 产品：`moo-chrome-dev-tool`，Chrome 扩展（Manifest V3），前后端业务调试工具——网页里发现 bug → 截图录屏 + 抓现场（网络请求 / console 错误 / DOM / storage 快照）→ 上报后端
- 团队：**mooeen**，中文「沐恩」，意为「沐浴在上帝的恩典」（基督教背景）
- **鹰图腾不是装饰**：对应圣经 以赛亚书 40:31「他们必如鹰展翅上腾」——鹰 = 锐利眼光 + 上腾更新 + 属灵力量。**鹰是身份核心，不可替换**为虫子、光标、箭头等纯工具符号
- 现有 icon（`public/icons/icon-128.png`）是「深圆底 + 写实鹰头 + 金色眼睛」的写实风照片，要换掉它但保留视觉记忆

## 2. 视觉方向（已确定，请勿翻盘）

- **类型**：鹰眼准星型 logo（双语义：锐利洞察 + 调试瞄准）
- **笔触**：实心剪影（filled silhouette），**不要** 线性 outline 风
- **构图**：3/4 侧鹰头（头部略转，单眼正对观众，喙朝侧前方）；**不要** 正面对称纹章风、**不要** 纯侧面 profile
- **容器**：圆形深色 badge 底；**不要** 透明方形裸鹰（作为主 icon 的话）
- **准星形态**：完整十字 reticle（外圈圆环 + 十字短线 + 中心瞳孔），整体落在那只可见的眼睛上——眼睛 = 瞄准镜

## 3. 配色（已确定）

| 元素              | 参考值     | 说明                                         |
| ----------------- | ---------- | -------------------------------------------- |
| BG 圆底           | `#2a2a2a`  | 暖深灰，可在 `#1a1a1a ~ #2a2a2a` 间微调      |
| 鹰剪影            | `#0d0d0d`  | 近黑                                         |
| 金色（虹膜+准星） | `#f4b400`  | 琥珀金，神性之光的隐喻，呼应"沐恩"           |

**全图唯一暖色亮点 = 那只眼睛**，鹰身不要再加其他颜色或渐变。

## 4. 形态硬性要求

鹰头剪影必须读得出以下四个解剖特征（**缺一不可**）：

1. **钩喙 J 形**：明确的下伸尖端 + 末端微微勾回，不是直喙
2. **眉骨悬岩**：眼睛上方有明显凸出 ledge，眼窝是凹陷——这是「凶猛凝视」的关键
3. **羽冠**：3-4 片，**高低错落**（不要等高城堡冠），朝头后倾斜，尖锐而非圆润
4. **下颌斜收**：下巴线条往后斜收，不要水平直边

气质：锐利、骨感、凝视感，**不是**可爱、不是圆润、也不要狰狞张牙。

视觉焦点：金色眼睛是唯一焦点，剪影的所有线条都应该把视线引向那里。

## 5. 交付物清单

- [ ] master SVG 源文件（`viewBox="0 0 512 512"` 或等比方形）
- [ ] PNG 栅格：**16 / 32 / 48 / 128**（这是 Chrome 扩展 manifest 必需的四档），加 256（用于 README / Chrome 商店）
- [ ] 透明背景的裸鹰头 monogram 版本（无圆底，用于 README 头图、文档插图）
- [ ] （可选）mooeen wordmark 字标（鹰 + 文字横排组合）

## 6. 验收标准

- **16×16 工具栏验收**（关键）：在 Chrome 工具栏的**浅色和深色主题**下都必须清晰——「深色圆底 + 金色眼睛点」组合在任何背景下都不能消失。鹰解剖细节允许在此尺寸损失，但圆底边界和金色亮点必须稳。
- **128×128 名片验收**：钩喙、眉骨、羽冠、下颌四个特征一眼可识别，整体像鹰不像「黑色斑点」。
- SVG path 控制点建议 ≤ 60（保持矢量轻量，扩展包大小友好）。

## 7. 反例（避免）

- ❌ 美国国徽风纹章鹰（太政治、太硬、太对称）
- ❌ 卡通可爱小鹰（违背调试工具定位）
- ❌ 写实照片处理风（现有 `eagle-source.jpg` 这类，小尺寸糊）
- ❌ 剪影上画羽毛纹理细节（128 看着丰富，16 立刻糊掉）
- ❌ 线性 outline 鹰头（已经决策不走这条线）

## 9. 资源接入方式

设计师交付后，PNG 替换路径：`public/icons/icon-{16,32,48,128}.png`。

如果想保留「源图 → 生成多尺寸」的脚本工作流，看 `scripts/generate-icons.mjs`——目前从 `src/assets/eagle-source.jpg` 出 4 档 PNG + 圆形 alpha mask。如果设计师直接交付分尺寸 PNG，可以跳过这个脚本；如果交付的是单张 master，需要更新脚本里的源文件路径。
